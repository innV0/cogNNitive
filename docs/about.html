About cogNNitive

The monorepo that ties the iNNv0 FORMAT ecosystem together.

Architecture

cogNNitive is organized as an npm workspaces monorepo with two main areas:

cogNNitive/
├── apps/
│   ├── format-editor/  // Vue 3 unified workspace editor (FILE + FOLDER)
│   └── launcher/        // Legacy — being consolidated into format-editor
├── packages/
│   └── format-core/ // TS parser, resolver, validator
├── specs/            // Specification files
├── models/           // Example models
└── docs/            // This website

Apps

@innv0/format-editor v0.1.0

format-editor

Unified Vue 3 workspace editor with Pinia state management and Vue Router. Uses the File System Access API to open workspace folders, runs a single recursive parse pass over every directory, and renders FILE and FOLDER mode models in one mixed tree. Features a sidebar tree navigator, metamodel-driven NodeForm, and IndexedDB handle persistence for fast reopen.

@innv0/launcher v0.1.0 — Legacy

cogNNitive Launcher

The original Vue 3 drag-and-drop app that detected FILE vs FOLDER mode and routed to separate editors. Now being consolidated into format-editor — its validation, history, and toast components are being ported over. Still available during the transition.

Package

@innv0/format-core v0.1.0

format-core

Framework-agnostic TypeScript library with a unified parser for FILE and FOLDER modes, model types, a validator against template schemas, IO drivers for both modes, and a parent-spec-chain resolver.

Specifications

The ecosystem defines four specification levels, each building on the previous:

Level 0

defiNNe

Meta-specification: structure, SemVer, RFC 2119. The root of the chain.

Level 1

FORMAT

Central specification with FILE (single document) and FOLDER (node-as-folder) modes.

Level 2

Templates

business, procedures, kb — each declares concepts, markers, and relationship types.

Level 3

Models

Concrete instances like Ghostbusters or TeamKB. Lightweight — just data and parent pointer.
