What is cogNNitive?

A monorepo that ties together the iNNv0 specification ecosystem: the unified format-editor, the format-core library, and all specs.

format-editor

Unified Vue 3 workspace editor. Open any folder, parse FILE and FOLDER mode models into a single tree, edit nodes with metamodel-driven forms, and validate against FORMAT specs.

@innv0/format-core

Shared TypeScript library — parser, resolver, and validator used by the format-editor. Framework-agnostic with IO drivers for both FILE and FOLDER modes.

Spec Chain

Four levels: defiNNe (meta) → FORMAT (spec) → Templates (business, procedures, kb) → Models. Each level adds structure over the last.

How format-editor works

Three steps from folder to editable tree.

1

Open workspace

Pick a folder via the File System Access API. The editor stores the handle in IndexedDB for fast reopen.

2

Single parse pass

The recursive parser walks every directory. _FORMAT.md directories become element nodes; bare directories become concept nodes. One tree for FILE and FOLDER together.

3

Edit & validate

Select any node in the sidebar tree and edit its fields in the metamodel-driven form. Run FORMAT compliance validation against spec contracts.

Ecosystem Architecture

The four-level spec chain that powers every FORMAT model.

Level 0

defiNNe

The meta-specification. Defines structure, versioning, and RFC 2119 conventions for the entire ecosystem.

Level 1

FORMAT

The central specification. Two modes: FILE (single document) and FOLDER (node-as-folder with assets). Unified relationship system.

Level 2

Templates

Domain-specific templates: business, procedures, kb (knowledge base). Each declares concepts, markers, and matrices.

Level 3

Models

Concrete data instances like Ghostbusters (business) or TeamKB (knowledge base). Lightweight — just data and parent pointer.

Sample Models

Four ready-to-open FORMAT models — one per template — that ship with the repo. Open any of these folders in the format-editor to see the tree, edit nodes, and validate against the spec chain.

business • FILE mode

Ghostbusters

A complete business model for a fictional ghost-catching franchise. Covers analysis (SWOT, risks, assumptions), business idea, objectives, market segments, finance, legal, and operations. Single-file _FORMAT.md with the full _F index tree.

specs/business_V_0-1-1/samples/Ghostbusters_V_0-1-2_business_FORMAT.md

catalog • FOLDER mode

Music History

Multi-directory catalog with 4 genres (Rock, Pop, Jazz, Electronic), 6 artists (The Beatles, Queen, Michael Jackson, Miles Davis, Kraftwerk), and 5 albums — each node is a folder with its own _FORMAT.md. Demonstrates the full FOLDER-mode tree structure with nested concepts.

specs/catalog_V_0-1-2/samples/Music_History_V_1-0-0_catalog/

procedures • FILE mode

Code Review Process

Structured procedure document for PR-based code reviews. Defines roles (Author, Reviewer, Maintainer), step-by-step workflow with tool bindings, artifacts (checklist, report), and an expedited hotfix path. Single _FORMAT.md with _F Procedure, _F Roles, and _F Work sections.

specs/procedures_V_0-1-1/samples/CodeReviewProcess_V_1-0-0_procedures_FORMAT.md

kb • FOLDER mode

Team Knowledge Base

FOLDER-mode knowledge base model. A TeamKB root node ready to be populated with Persona, Topic, and Reference concept folders — each a directory with metadata and physical assets. The starting point for building a team-wide documentation base using the KB template.

archive/2026-07-02/models/TeamKB_V_0-1-1_kb/
