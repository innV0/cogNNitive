# Business Template

## A comprehensive template for business strategy modeling with 70+ concepts covering market, stakeholders, value propositions, operations, finance, and analysis

## Philosophy

The Business Template is designed for modeling complete business strategies in a structured, interconnected way. It follows the belief that a business model is best understood as a network of relationships — between problems and solutions, stakeholders and channels, assumptions and risks — rather than as a linear document.

## Objectives

- Provide a comprehensive set of concepts covering all major business domains: market, strategy, operations, finance, team, legal, and validation.
- Enable relationship mapping between concepts via evaluable matrices (Problems↔Value Propositions, Assumptions↔Risks, etc.).
- Support both narrative (text-type) and scored (weight-type) concepts for mixed qualitative/quantitative modeling.
- Serve as the default template for business model authoring in the FORMAT ecosystem.

## Specification

### 1. Supported Mode

This template supports **FILE mode only**.

### 2. Concepts

The template defines 70+ concepts organized into these categories:

| Category | Concepts | Purpose |
|---|---|---|
| **Market** | Stakeholders, Segments, Profiles, Competition, Trends, Size | External environment |
| **Value** | Problems, Value Propositions, Messages, Channels | Value delivery chain |
| **Solutions** | Products, Services, Features, Roadmap, Pricing | Offerings |
| **Marketing** | Branding, Naming, Media Plan, Pitch, Web | Go-to-market |
| **Organization** | Mission, Vision, Values, Goals, Team | Internal structure |
| **Operations** | Activities, Functions, Resources | Execution |
| **Finance** | Revenue, Costs, Funding, Projections | Economics |
| **Analysis** | Assumptions, Risks, SWOT, Keys | Strategic review |
| **Validation** | Coherence, Experiments | Testing |

Each concept has a `type` that determines its representation:

| Type | Behavior |
|---|---|
| `text` | Single-instance, free-form Markdown |
| `weight` | Multi-instance list with numeric weight marker |
| `category` | Grouping concept for sub-concepts |
| `steps` | Ordered sequence of steps |
| `sequence` | Ordered list of events |

### 3. Markers

| Marker | Symbol | Purpose |
|---|---|---|
| `weight` | `*` | Core importance score (1–10) |
| `completion` | `>` | Progress indicator |
| `certainty` | `?` | Confidence level (1–5) |
| `priority` | `!` | Urgency flag |
| `rating` | `+` | Quality rating |

### 4. Matrices

The template declares the following evaluable matrices:

| Matrix | Source → Target | Purpose |
|---|---|---|
| Journey Map | Journey → Emotions | Customer experience mapping |
| Problems-Value Propositions | Problems → Value propositions | Solution fit |
| Value propositions-Messages | Value propositions → Messages | Message alignment |
| Messages-Channels | Messages → Channels | Channel strategy |
| Assumptions-Risks | Assumptions → Risks | Risk assessment |
| Features-Milestones | Features → Milestones | Release planning |

All evaluable matrices use a 9-point scale: `Max;Very High;High;Slightly High;Neutral;Slightly Low;Low;Very Low;Min`.

### 5. Relationship Types

| Type | Enabled | Representation |
|---|---|---|
| Hierarchy | ✅ | index block (wikilinks) |
| Evaluable matrix | ✅ | Source→target tables |
| Graph edge | ❌ | Not applicable |
| Sequence | ✅ | concept type `steps` / `sequence` |

### 6. Taxonomy Index

Concepts are organized in a multi-level hierarchy. The root categories are:

```
Market → Segments → Profiles → Persona ...
Solutions → Products and services → Features ...
Organization → Team → Goals, Skills, Positions ...
Finance → Revenue, Costs, Funding sources ...
Analysis → Assumptions, Risks, SWOT, Keys ...
Validation → Coherence, Experiments ...
```

```yaml
---

> [!NOTE]
> This is a **FORMAT document**...

# _F index
* [[Market]]
  * [[Stakeholders]]
  * [[Segments]]
    * [[Problems]]
    * [[Value propositions]]
...

# _F Business summary
Your business summary here.

# _F stakeholders
* _F Stakeholders: Stakeholder Name
  Description and details.

# _F matrices: problems-value propositions matrix
| Problems \ Value propositions | VP1 | VP2 |
| :--- | :---: | :---: |
| Problem 1 | Max | High |
```

The application will resolve the `parent` URL, download this template, and use its concepts/markers/matrices to validate and render your model.
